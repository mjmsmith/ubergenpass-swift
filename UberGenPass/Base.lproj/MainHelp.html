Site

Type or paste the site that you want to generate a password for into the Site field. You can enter either a URL ("http://www.example.com/x?y=z") or a bare domain name ("www.example.com"). If you've enabled the Save Recent Sites setting, recent sites that match the typed text are displayed in a popup list. Tap on an item in the list to select it.

Since a URL is typically copied and pasted from Safari, UberGenPass applies the default behavior of removing subdomains. The following URLs all generate the password for "example.com":

  • http://example.com
  • http://www.example.com/index.html
  • http://xxx.yyy.example.com/zzz

On the other hand, since a bare domain name is typically typed directly, UberGenPass assumes that you want a password for the full domain as typed. Using the domains in the URLs above, a (different) password would be generated for "example.com", "www.example.com", and "xxx.yyy.example.com" respectively.

Password Length

Tap + or - to set the desired number of characters in the generated password. This setting is persisted between sessions.

Password Type

Tap MD5 or SHA to select the type of password generated. This setting is also persisted bwetwen sessions.

Notes

As soon as the Site field contains a URL or domain, UberGenPass displays the domain name and its generated password behind "•••" bullet characters. To display the plain password text, tap on the bullet characters.

The generated password is automatically updated whenever you change the site text, password length, or password type; you don't have to tap the Done key on the keyboard to make it happen. To hide the keyboard, tap the Done key or tap anywhere on the background.

When a password is generated, two buttons are available below it.

Copy to Clipboard

Tap Copy to Clipboard to copy the password text to the clipboard. A checkbox appears next to this button when the current password is equal to the current clipboard contents.

Launch Safari

Tap Launch Safari to launch Safari at the URL or domain in the Site field. If the corresponding page is already open in Safari, it normally returns you to that open page rather than loading the URL in a new page.
